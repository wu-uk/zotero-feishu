# Zotero 飞书同步

[English](../README.md)

Zotero 飞书同步是一个非官方 Zotero 插件，用于将文献元数据、笔记、图片和
PDF 附件发布到飞书云文档。Zotero 始终是数据源；再次同步时，插件会替换已
映射飞书文档中由插件管理的内容。

## 使用要求

- Zotero 8-9；发布前验证以 Zotero 9 为准。
- 中国大陆飞书账号，并且能够创建个人应用和授予所需权限。组织策略可能要求
  管理员审批。
- 同步时，Zotero 必须能够访问本地 PDF 和笔记图片文件。

## 安装

1. 从最新 GitHub Release 下载 `zotero-feishu-sync.xpi`。
2. 在 Zotero 中打开 **工具 → 插件**。
3. 选择**从文件安装插件**，打开 XPI，并在 Zotero 提示时重启。

## 连接飞书

1. 打开 **Zotero 设置 → Zotero Feishu Sync**，点击**连接飞书**。
2. 在浏览器中确认创建专用个人应用。
3. 如果打开飞书权限管理页，请开通列出的权限，并完成组织要求的应用发布或
   管理员审批。
4. 在浏览器中完成账号授权，然后返回 Zotero。

不需要手动填写 App ID、App Secret 或重定向地址。同步位置留空时使用“我的
空间”根目录，也可以填写飞书云空间文件夹链接或 Token。

## 使用方法

- 右键点击普通文献、文献的子笔记或子附件，选择
  **飞书同步 → 同步选中文献**。
- 右键点击分类，选择**同步当前分类到飞书**。
- 在相同菜单中打开或明确删除条目对应的飞书文档。
- 在飞书状态列查看同步中、成功或失败详情。

每篇普通文献对应一个飞书 Docx。插件确认映射文档仍存在后，会跳过内容没有
变化的条目。子笔记会保留与 Markdown 对应的结构，笔记图片会被上传，本地
子 PDF 会被追加到文档末尾。

## 当前限制

- 仅支持“我的文库”和中国大陆飞书。
- 不支持独立笔记和独立附件；选择文献下的子笔记或子附件时会同步其父文献。
- PDF 标注必须先添加到 Zotero 笔记中。
- 不会把飞书中的编辑导回 Zotero，下一次同步可能覆盖这些编辑。
- 条目映射只保存在本机 Zotero 配置目录中；映射丢失后可能为同一条目创建
  第二份飞书文档。

## 隐私与移除

文献内容由 Zotero 直接发送到飞书，不经过开发者运营的中转服务器。应用凭据
和 OAuth Token 保存在 Zotero 的 Firefox Login Manager 中；条目映射保存在
`<Zotero 配置目录>/zotero-feishu/state.json`。

断开连接会删除 OAuth Token，但会保留自动生成的应用凭据，以便下次重新连接。
卸载插件不会删除飞书中的个人应用或已同步文档；不再使用时，请在飞书开放平台
和云空间中手动删除。

## 开发

```bash
npm install
npm start
npm run lint:check
npm run build
npm test
```

将 `.env.example` 复制为 `.env`，并使用专用 Zotero 开发配置目录。生产 XPI 和
更新清单位于 `.scaffold/build/`。
